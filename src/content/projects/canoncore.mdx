## The Product

CanonCore turns Google Drive into a fully featured media library: visual browsing, metadata enrichment, progress tracking, and public sharing, without moving or duplicating your files. The name comes from "canon" as the app pulls canonical metadata from sources like [TMDB](https://www.themoviedb.org). Create folder structures for movies and TV shows, reorganise with drag and drop, enrich items with posters and backdrops, track what you've watched, and share collections publicly. Your files live in your storage, not ours.

I use this daily for my own movie and TV library. Active development continues with a React Native iOS app and interactive 3D graph visualisation (see [In Progress](#in-progress) below).

## How It's Built

### Architecture Decisions

**Google Drive migration:** I originally built this on SFTP, but path-based matching meant every rename or move created duplicates. No stable IDs, no change detection API, read-only from the web. Google Drive solved all of it: permanent file IDs survive renames and moves, Changes API for incremental sync, full read/write access so users can create folders directly from CanonCore. Should have started here.

**Stack Auth → NextAuth.js v5:** I started with managed auth, then migrated to self-hosted JWT sessions after hitting rate limits. I added **23+ rate limiters** via [Upstash Redis](https://upstash.com) with different thresholds per action (strict for auth, generous for browsing).

**Four-layer state management:** Global client state lives in Redux Toolkit (media player queue, playback position). Server state uses TanStack React Query (items, playlists, metadata). URL state uses nuqs (search filters, pagination cursors). Local component state stays in useState. Each layer has a single responsibility, so nothing competes or duplicates.

---

### Security & Resilience

Account lockout after 5 failed login attempts (15-minute progressive delay). JWT token versioning invalidates all sessions on password change. Email verification for signup and email changes. OWASP security headers on every route: CSP, HSTS with preload, X-Frame-Options DENY, and Permissions-Policy. All database mutations are logged via a Prisma Client Extension with automatic redaction of sensitive fields (passwords, tokens, credentials). Circuit breaker on [TMDB](https://www.themoviedb.org) and Google Drive calls to handle failures gracefully. Edge-level bot protection blocks **37 AI crawlers** while rate-limiting beneficial bots. Zod schema validation across all server actions.

---

### Performance

Extracted a view-only Grid from SortableGrid to avoid [dnd-kit](https://dndkit.com) overhead in browse mode. Google Drive operations batched up to **100 per request**, reducing sync time from **~45s to ~3s**. Video streaming uses HTTP 206 range requests with 10MB default chunks for instant seeking. Virtualised lists via [TanStack Virtual](https://tanstack.com/virtual) for large collections. Cursor-based keyset pagination throughout (no offset queries). Colour extraction runs server-side via [sharp](https://sharp.pixelplumbing.com) and feeds 10 CSS custom properties registered with `@property` for animated colour transitions on hero pages. `content-visibility: auto` on grid cards and tree items for off-screen rendering performance.

---

### Accessibility

WCAG 2.1 AA compliant throughout. Reduced motion support via a custom hook that disables carousel autoplay and animations. Skip link to main content. ARIA live regions for drag-and-drop announcements.

---

### Testing

**3,000+ tests** across unit, integration, and E2E layers. [Vitest](https://vitest.dev) unit tests cover auth, items, Google Drive sync, and crypto operations. Integration tests run against real [Neon](https://neon.tech) PostgreSQL. E2E tests with [Playwright](https://playwright.dev) use **16 Page Object Models** and test against a real Google Drive account, not mocked. Tests automatically create missing fixtures for self-healing reliability. **69 Storybook stories** with [axe-core](https://github.com/dequelabs/axe-core) accessibility testing that fails the build on WCAG 2.1 AA violations.

---

### Observability

[Sentry](https://sentry.io) error tracking across client, server, and edge runtimes with source maps for readable stack traces. Layered error boundaries catch failures at each route scope and report to Sentry automatically. [OpenTelemetry](https://opentelemetry.io) distributed tracing via Vercel's integration and Speed Insights for Core Web Vitals monitoring in production.

---

### CI/CD

GitHub Actions pipeline runs a quality gate (format, lint, type check, unused code detection) then kicks off tests and production build in parallel. Schema migrations deploy across **4 Neon database branches** (development, production, demo, seed) with automated seeding after each migration. A separate Storybook CI workflow builds and runs interaction tests with Playwright. [Husky](https://typicode.github.io/husky) pre-commit hooks run ESLint and Prettier on staged files. Conventional commits enforced by commitlint.

---

### SEO & Social Sharing

Dynamic OpenGraph images generated server-side for every public profile and item page. Shared links show rich preview cards on Twitter, Discord, and Slack. JSON-LD structured data (Movie, TVSeries, Person schemas) for search engines. Dynamic sitemap keeps all public profiles and items indexed.

**69 component stories** documented in [Storybook](https://canoncore-storybook.vercel.app) with [axe-core](https://github.com/dequelabs/axe-core) accessibility checks, [MSW](https://mswjs.io) API mocking, and custom viewports (iPhone, iPad, Desktop, Desktop XL).
